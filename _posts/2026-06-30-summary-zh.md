## [最高法院裁定地理围栏搜查令需受宪法保护](https://www.theguardian.com/us-news/2026/jun/29/supreme-court-geofence-warrants-case-decision) ⭐️ 9.0/10

## [将 Game Boy 指令 JIT 编译为 WASM，性能超越原生解释器](https://humphri.es/blog/WATaBoy/) ⭐️ 8.0/10

一篇博客文章展示，将 Game Boy 模拟器指令 JIT 编译为 WebAssembly（WASM）可以超越原生解释器的性能，通过利用浏览器优化的 WASM 运行时获得更高性能。 这种方法提供了一种绕过平台 JIT 限制（例如 iOS）的新方式，通过使用浏览器的 WASM 引擎，可能使高性能模拟在以前受限的平台上成为可能。 该项目名为 WATaBoy，在运行时将 Game Boy CPU 指令编译为 WASM 模块。观察到 Firefox 比 Chrome/Safari 慢 25%，可能是由于 WASM JIT 编译策略的差异。

**背景**: 模拟器传统上使用解释执行或原生 JIT 编译。iOS 上 JIT 编译受到严格限制，但浏览器可以为 JavaScript 和 WebAssembly 使用 JIT。通过以 WASM 为目标，模拟器可以利用浏览器的 JIT 引擎，实现比纯解释执行更高的性能。

## [深入解析：CUDA 内核从 CPU 到 GPU 的启动路径](https://fergusfinn.com/blog/what-happens-when-you-run-a-gpu-kernel/) ⭐️ 8.0/10

Fergus Finn 发表了一篇详细的博客文章，从头到尾讲解了启动 CUDA 内核的完整过程，从主机 CPU 代码到 GPU 的 warp 调度器，涵盖了驱动交互、通过 doorbell 和队列管理描述符（QMD）提交命令，以及在流多处理器上的 warp 调度。 这篇文章填补了典型 CUDA 教程中的一个关键空白——通常教程只讲到内核语法和 warp 执行，而忽略了 CPU 到 GPU 的路径。理解这一路径有助于开发者在高性能计算和 GPU 编程中优化性能并调试问题。 文章解释了 doorbell 机制和 QMD 格式，它们将 CUDA 启动语法与实际提交给 GPU 硬件的内容联系起来。文章还详细说明了 GPU 的 warp 调度器如何在每个时钟周期内选择符合条件的 warp 来执行。

hackernews · mezark · 6月29日 13:11 · [社区讨论](https://news.ycombinator.com/item?id=48718863)

**背景**: CUDA 是 NVIDIA 的并行计算平台，允许开发者使用 GPU 进行通用计算。内核是在 GPU 上运行的函数，启动内核涉及 CPU 向 GPU 驱动发送命令，驱动再将工作提交给硬件。GPU 以 32 个线程为一组（称为 warp）执行，warp 调度器决定接下来运行哪个 warp。

## [Qwen 3.6 27B：强大的本地模型，高昂的硬件成本](https://quesma.com/blog/qwen-36-is-awesome/) ⭐️ 7.0/10

Qwen 3.6 27B 是一款新发布的开源 27B 参数模型，因其强大的编码和智能体能力而受到赞誉，性能可与十倍于其规模的模型相媲美。然而，本地运行它需要昂贵的硬件，例如 128GB MacBook Pro，价格超过 6,699 美元。 这凸显了本地隐私与成本效益之间的持续权衡，因为云 API 以极低的硬件投资提供类似或更好的模型。这场争论影响着那些决定是投资高端本地设备还是依赖云服务的开发者。 该模型擅长智能体编码和仓库级推理，并引入了“思维保留”功能以在迭代中保留推理上下文。然而，在笔记本电脑上运行会导致明显的发热和噪音，使其不适合实际操作。

hackernews · stared · 6月29日 17:05 · [社区讨论](https://news.ycombinator.com/item?id=48721903)

**背景**: 本地 LLM 需要大量 VRAM（Mac 上的统一内存）才能高效运行大型模型。对于 27B 参数模型，建议至少 64GB 统一内存，128GB 则更为理想。Apple Silicon Mac 使用统一内存，同时充当 RAM 和 VRAM，使得在消费级硬件上进行大型模型推理成为可能。
