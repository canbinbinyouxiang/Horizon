## [WATaBoy 将 Game Boy 指令 JIT 编译为 WebAssembly，性能超越原生解释器](https://humphri.es/blog/WATaBoy/) ⭐️ 8.0/10

WATaBoy 是一个本科毕业设计项目，它实现了一个 JIT 编译器，将 Game Boy 的 SM83 指令翻译为 WebAssembly，性能比原生解释器更快。该项目被构建了两次——一次作为纯解释器，一次采用 JIT 到 Wasm 的方法——并通过基准测试展示了性能提升。 这项工作展示了一种巧妙利用浏览器 JIT 能力进行模拟的方法，可能使得在限制传统 JIT 编译的平台（如 iOS）上运行高性能模拟器成为可能。它还表明，由于解释的巨大开销与 Wasm 相对较小的开销相比，编译为 WebAssembly 可以击败原生解释。 该 JIT 编译器在运行时将 Game Boy SM83 指令翻译为 WebAssembly 模块，然后由浏览器的 Wasm 引擎进一步 JIT 编译为原生代码。基准测试显示，JIT 到 Wasm 的方法优于原生解释器，在此工作负载下 Firefox 比 Chrome/Safari 慢约 25%。

hackernews · energeticbark · 6月29日 15:02 · [社区讨论](https://news.ycombinator.com/item?id=48720190)

**背景**: JIT（即时编译）在运行时将代码编译为原生机器码以加快执行速度，但苹果在 iOS 上限制 JIT，除了浏览器的 JavaScript 和 WebAssembly 引擎。WebAssembly 是一种低级二进制格式，在浏览器中以接近原生的性能运行。传统模拟器通常使用解释执行，这比 JIT 编译慢。

## [Outer Shell：SSH 原生图形化外壳](https://probablymarcus.com/blocks/2026/06/28/native-graphical-shell-for-SSH.html) ⭐️ 7.0/10

Marcus Lewis 发布了 Outer Shell，这是一个开源的 SSH 原生图形化外壳，能够将 Jupyter 和 Tensorboard 等远程 Web 应用通过 SSH 隧道传输到本地浏览器，无需手动进行 SSH 端口转发。 这简化了远程开发工作流程，为访问远程服务器上的图形化应用提供了无缝的浏览器界面，解决了数据科学家和开发者的常见痛点。 Outer Shell 使用基于套接字的 Web 服务器通过 SSH 代理 HTTP 请求，并设计为可与任何监听本地端口的 Web 应用配合使用。该项目是开源的，可在 GitHub 上获取。

**背景**: SSH 隧道是一种通过加密的 SSH 连接传输任意网络数据的方法，常用于安全访问远程服务。传统上，访问 Jupyter 等远程 Web 应用需要手动使用 'ssh -L' 命令设置 SSH 端口转发。Outer Shell 通过提供图形界面来自动管理隧道并在本地浏览器中启动应用，从而简化了这一过程。

## [桑迪亚国家实验室 SA3000：1970 年代的抗辐射 8085 处理器](https://www.cpushack.com/2026/06/03/sandia-national-labs-sa3000-8085-cpu/) ⭐️ 7.0/10

这凸显了美国早期在国产抗辐射芯片制造上的政府投资，与如今依赖承包商形成对比，并为当前用于太空和国防的抗辐射处理器提供了历史背景。 SA3000 采用 n-on-n+外延衬底以控制闩锁效应，使用了大量保护环和硬化氧化物。其性能远超设计目标（1×10^5 rads），在 1×10^6 rads 下仅损失 25%性能。

hackernews · rbanffy · 6月29日 10:20 · [社区讨论](https://news.ycombinator.com/item?id=48717287)

**背景**: 抗辐射加固是使电子设备抵抗电离辐射的过程，对太空和核应用至关重要。Intel 8085 是 1977 年推出的 8 位微处理器。桑迪亚国家实验室是美国政府专注于核安全的研究实验室。
