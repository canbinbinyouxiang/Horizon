SpudCells use a simple membrane and a synthetic division machinery derived from bacterial and viral genes, bypassing the complex cytoskeleton. The cells can feed, metabolize, replicate DNA, and divide, passing genetic material to offspring.

SpudCell's design eliminates this need, simplifying the process.

## [FFmpeg 9.1 Introduces New AAC Encoder with Caveats](https://hydrogenaudio.org/index.php/topic,129691.0.html) ⭐️ 7.0/10

FFmpeg 9.1 includes a new native AAC encoder that offers improved audio quality compared to the previous FFmpeg AAC encoder, but it is limited to constant bitrate (CBR) mode and optimized primarily for 48 kHz sampling rate. This update is significant because FFmpeg is a widely used open-source multimedia framework, and its previous AAC encoder was known for poor quality and artifacts. The new encoder provides a better built-in option, reducing reliance on external encoders like Apple's Core Audio. The encoder only supports CBR mode and is optimized for 48 kHz; other sample rates like 44.1 kHz will work but may not achieve the best quality. Additionally, the encoder works around a stereo PNS bug in FFmpeg's AAC decoder, which had gone unnoticed for years.

**Background**: AAC (Advanced Audio Coding) is a lossy audio compression standard designed to succeed MP3, offering better quality at similar bitrates. FFmpeg is a popular open-source tool for handling multimedia files, and its native AAC encoder has historically been criticized for poor quality compared to alternatives like libfdk_aac or Apple's Core Audio encoder.

## [IPFS Content Publishing 10x Faster via Async PUT RPCs](https://probelab.io/blog/optimistic-provide/) ⭐️ 7.0/10

Probelab 通过将大部分 PUT RPC 改为异步，在大多数（而非全部）PUT RPC 成功后即返回控制权给用户，并在后台继续处理剩余请求，从而使 IPFS 内容发布速度提升高达 10 倍。 This improvement significantly reduces the time users wait for content to be published on IPFS, making the system more practical for real-world applications. However, the trade-off between completeness and speed raises questions about reliability in production environments. The approach does not actually perform the same work faster; it reduces the amount of synchronous work by deferring some PUT RPCs to the background. This means the content may not be fully published to the DHT immediately, which could affect discoverability.

hackernews · dennis-tra · Jul 1, 15:30 · [Discussion](https://news.ycombinator.com/item?id=48748518)

**Background**: IPFS (InterPlanetary File System) is a decentralized peer-to-peer file system that uses content-addressing to store and share files. Content publishing involves announcing the availability of a file to the network via a Distributed Hash Table (DHT), which requires multiple PUT RPCs to peers. Traditionally, these RPCs are synchronous, causing delays as the client waits for confirmations.

Astryx has been used internally at Meta for eight years, powering over 13,000 apps, and its open-source release includes a CLI and MCP server to make components easily consumable by AI agents.

**Background**: A design system is a collection of reusable components and guidelines that ensure visual and functional consistency across applications. Agent-ready interfaces are UIs designed to be understood and operated by AI agents, often via protocols like MCP (Model Context Protocol).
